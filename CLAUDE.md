### 1.4 Mission & Core Strategy
- **Mission**: 케어의 효율성·정확성·투명성 확보를 위한 데이터 기반 운영 체계 구축 → 어르신과 종사자 모두가 행복한 환경.
- **Core Strategy 3가지**:
  - **Data-Driven**: 현장의 실시간 데이터를 통한 정교한 운영
  - **Tech Integration**: AI·로봇 기술을 실제 요양 환경에 최적화하여 통합
  - **Human-Centric**: 절감된 리소스를 종사자 처우·서비스 질로 환원

### 1.5 핵심 솔루션 — 돌보라 OS 3-Step
> *"추가 장비 도입 없이, 소프트웨어 업데이트만으로 시작하는 스마트 요양원."*

| Step | Keyword | 내용 |
| :-- | :-- | :-- |
| **Step 1** | **Safety** | **Edge-AI 비전 엔진** — 기존 CCTV를 ‘눈’으로 활용. 관절 각도·자세 변화·체위 유지 시간 분석. 낙상 및 야간 이상 행동 실시간 탐지. |
| **Step 2** | **Freedom** | **자동 케어 기록 시스템** — 기저귀 교체·이동 보조 등 케어 활동을 AI가 감지해 디지털 로그 생성. RPA 방식으로 기존 ERP 자동 입력 보조. |
| **Step 3** | **Trust** | **통합 안심 대시보드** — 활동량·수면 패턴·운영 지표를 데이터화. 시설장에게는 리스크 관리, 보호자에게는 실시간 안심 리포트. |

### 1.6 사업 영역 (3-Phase 로드맵)
- **Phase 1 (현재)** — Vision AI 기반 안전 & 자동 기록 (기존 CCTV 활용)
  - 하이엔드 시니어 케어 설계 (밸류업 컨설팅)
  - AI/로봇 통합 솔루션 (한국형 스마트 요양 시스템 보급)
  - R&D 파트너십 (국가 연구기관 협업)
- **Phase 2 (도약)** — 레이더 및 정밀 센싱 통합 (정확도 99% 목표)
  - 다중 복합 센싱 (Multi-Modal)
  - 레이더 센싱 모듈 (호흡·맥박·미세 움직임 비접촉 측정)
  - 데이터 교차 검증 (낙상 오탐지율 감소)
- **Phase 3 (미래)** — 케어 로봇 및 AI 비서 통합
  - 이송·배설 보조 로봇 (요양보호사 근골격계 보호)
  - 정서 케어·인지 로봇
  - 현장 리빙랩 검증

### 1.9 운영 필수 섹션 — FAQ (3개, 본문에 답변까지 포함)
1. **Q. 기존 CCTV 그대로 사용 가능한가요?**
   A. 네. 돌보라 OS는 하드웨어 독립적 소프트웨어 엔진. 기존 인프라 활용으로 추가 장비 부담 없음.
2. **Q. 사생활 침해 문제는 없나요?**
   A. Edge-AI로 영상 원본은 현장 즉시 분석 후 폐기. 식별 불가능한 스켈레톤·수치만 저장.
3. **Q. 요양보호사가 기기 사용 어려워하지 않을까요?**
   A. ‘자동화’ 중심. 별도 앱 조작이 아닌, AI가 작성한 기록을 ‘확인’만 하는 직관적 UI.

## 4. 기술 스택 & 디렉토리 구조

### 4.1 스택
- **Framework**: Next.js **14.2** (App Router) — `src/` 디렉토리 + TypeScript
- **Styling**: Tailwind CSS **3.4**
- **Animation**: framer-motion **11**
- **빌드**: 모든 페이지 **정적 prerender** (`○ Static`)
- **Node**: v20 / npm v9 환경에서 검증

2. `jaceyoon/DolboraWebsite` 저장소 선택 → **Import**
3. Framework Preset이 자동으로 **Next.js**로 인식됨. 환경변수 없음. 그대로 **Deploy** 클릭
4. ~1분 후 `dolbora-website-xxxx.vercel.app` 형태 URL 발급
5. 이후 `git push origin main` 할 때마다 자동 재배포

### 7.2 CLI 배포 (대안)
```bash
npx vercel        # 첫 실행 시 GitHub/이메일 로그인 안내
npx vercel --prod # 프로덕션 배포
```

### 7.3 커스텀 도메인 연결 (예: `dolbora.com`)
- Vercel Project → Settings → Domains에 도메인 추가
- 도메인 등록처(가비아·후이즈 등)에서 안내된 A/CNAME 레코드 입력 → 자동 SSL 발급

### 7.4 다른 옵션
- **Netlify**: 동일하게 GitHub 연동. Next.js 14 App Router 지원.
- **Cloudflare Pages**: 무료 트래픽 한도가 가장 큼. Next.js는 `@cloudflare/next-on-pages` 어댑터 필요.
- **AWS Amplify / Azure Static Web Apps**: 가능하지만 Vercel 대비 설정이 복잡.

원본 기획서의 핵심 통찰 3가지 (요양 현장의 ‘세 가지 비극’):
- **기록에 갇힌 돌봄 (Data Gap)** — 요양보호사 업무 시간의 30%+가 기록 작성에 소모
- **보이지 않는 위협 (Safety Blind-spot)** — 찰나의 순간에 발생하는 낙상·사고
- **운영의 블랙박스 (Transparency)** — 케어 활동을 데이터로 증명할 길이 없음

→ 돌보라 OS는 이 세 가지 비극을 **자동 기록 / Edge-AI 안전 감지 / 통합 대시보드**로 동시에 해결하는 솔루션입니다.
